3. STICKER DATA:
----------------
stickers/stickers_map.json
  - Maps sticker filenames to metadata
  - Format: { "filename.gif": { "emotion": "...", "description": "..." } }

stickers/GIFs/
  - Directory containing all sticker GIF image files
  - Referenced by stickers_map.json

API ENDPOINTS:
==============
GET /api/stickers
  - Returns: { "success": true, "stickers": [...] }
  - Each sticker: { "filename", "emotion", "description", "url" }

POST /api/stickers/validate
  - Requires: writer/editor authentication
  - Body: { "filename": "..." }
  - Returns: { "success": true, "filename", "emotion", "description" }

ROUTES:
=======
/static/stickers/*
  - Serves sticker GIF files
  - Maps to stickers/ directory

FRONTEND INTEGRATION:
=====================
1. Button in main panel calls showStickerPicker()
2. Modal loads stickers via /api/stickers
3. User selects sticker and role (User/Character)
4. selectSticker() sends POST to conversation API with:
   {
     "role": "User" | "Character",
     "content_type": "sticker",
     "sticker": {
       "filename": "...",
       "emotion": "...",
       "description": "..."
     }
   }
5. Sticker is displayed in conversation with image and metadata

DEPENDENCIES:
=============
- Go Chi router (github.com/go-chi/chi/v5)
- File serving capabilities
- JSON parsing
- Authentication middleware (for validate endpoint)
