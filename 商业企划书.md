## 二、核心竞争力：数据工厂模式

我们将数据收集过程重构为精密运转的工厂流水线：

### 2.1 生产车间 (wl2.studio)

不同于竞争对手的手工小作坊或合成数据，我们拥有完整的数字化生产平台：

| 生产环节 | 功能模块 | 作用 |
|---------|---------|------|
| **原料输入** | 对话编辑器 | 结构化录入界面，强制约束时间、日程、亲密度等7大维度 |
| **质量控制** | QC审核系统 | 编辑团队实时审核，不合格品直接驳回，保证良品率 |
| **工人管理** | 培训与认证 | 写手必须通过教程、考试、试稿三道关卡才能上岗 |
| **薪酬结算** | 自动计件系统 | 按通过字数自动结算，实时反馈收入，激励高产 |
| **社区生态** | 内部论坛/聊天室 | 形成"工友"文化，老带新，降低培训成本，提高留存 |

### 2.2 劳动力红利

我们成功利用了**大学生群体**作为核心劳动力，实现了极致的成本控制与质量平衡：

- **人力成本低**：相比全职标注员，大学生兼职成本极具优势（~0.1元/条）。
- **素质匹配高**：大学生本身是二次元/网文/游戏受众，理解角色扮演内核，无需从头教育。
- **管理社区化**：通过排行榜、社区交流、优秀作品展示，将枯燥的标注工作转化为具备成就感的创作活动。

## 三、商业模式与供应链

我们定位于**AI产业链的上游核心供应商**。

```
[ 上游：原材料 ]      [ 中游：加工厂 (我们) ]      [ 下游：客户 ]
大学生/兼职写手  ──►  wl2.studio 数据平台   ──►  OpenAI / 字节跳动 / 游戏厂商
(创意输入)           (标准化/质检/结构化)        (购买数据 / 接入模型API)
```

### 3.1 产品形态

我们生产的不仅是文本，而是**"4D数据"**（Text + Time + Context + Emotion）：

1. **B2B 数据集授权**：按条数/规模向大模型厂商出售原始结构化数据。
2. **Model-as-a-Service (MaaS)**：提供微调好的"持续性人格模型"API。
3. **定制化生产**：接受游戏厂商委托，生产特定IP（如《原神》角色）的专属数据。

### 3.2 收入预测

- **短期**：依靠数据授权快速回笼资金。
- **中期**：随着数据量级达到千万级，模型能力产生质变，API调用收入超越数据销售。
- **长期**：成为行业标准制定者，任何想要做"拟人AI"的公司都离不开我们的数据格式。

## 五、技术壁垒

我们的壁垒不只于"我们也懂AI"，而在于我们建立了**不可复制的数据生产SOP（标准作业程序）**。

### 5.1 数据形态认知壁垒

行业内大多数公司（包括大厂）还在盲目收集"一问一答"，而我们定义了"角色扮演"的标准数据格式：

```json
{
  "character_schedule": { "早晨": "...", "下午": "..." },  // 行为锚点
  "user_schedule": { "早晨": "...", "下午": "..." },       // 话题锚点
  "history": "...",                                      // 长期记忆锚点
  "relationship": "...",                                 // 关系锚点
  "dialogue": [
    {
      "role": "Kurisu",
      "content": "...",
      "timestamp": "14:30",  // 时间锚点
      "intimacy_delta": 0.05 // 情感锚点
    }
  ]
}
```

这7大维度的结构化数据，是OpenAI等公司目前极度缺乏且难以通过爬虫获取的（因为这是私密对话，且需要人工精心构造）。

### 5.2 平台技术壁垒

wl2.studio 并非简单的表单工具，而是集成了：
- **实时推理**：集成 vLLM + Qwen3-14B，写手可实时预览AI表现。
- **自动化流**：自动生成角色记忆档案，减轻写手负担。
- **多模态支持**：原生支持表情包系统，训练模型"发图"能力。
